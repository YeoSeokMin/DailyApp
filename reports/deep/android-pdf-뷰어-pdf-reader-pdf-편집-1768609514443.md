## 핵심 인사이트

### 개발자를 위한 교훈

1. **로컬 특화 전략의 양면성**: HWP 지원은 한국 시장에서 명확한 차별점이지만, 글로벌 확장과 정책 변화(ODF 전환)에 취약함. 로컬 특화와 글로벌 확장 사이 균형 필요. [추론]

2. **통합 앱의 가치**: 여러 기능을 하나의 앱으로 통합하면 사용자 편의성 증가하지만, 각 기능별 전문 앱(Adobe PDF, 한컴뷰어)과 비교 시 깊이가 부족할 수 있음. "올인원 vs 전문화" 딜레마. [추론]

3. **개발사 브랜딩**: "QR Code Scanner"라는 개발사명이 PDF 앱의 신뢰도를 떨어뜨릴 수 있음. 제품군에 맞는 브랜드 아이덴티티 중요. [추론]

4. **광고 모델의 한계**: 무료+광고 모델은 진입 장벽을 낮추지만, 깔끔 한글뷰어처럼 최소 광고 정책으로 높은 평점을 받는 경쟁자 대비 불리. 광고 빈도와 사용자 경험 균형 필수. [추론]

5. **기술 트렌드 대응**: 알PDF의 AI 요약 기능처럼 최신 기술(AI, 클라우드 연동) 탑재가 경쟁력 확보에 필수적. 단순 뷰어로는 차별화 어려움. [추론]

### 개선 제안

1. **AI 문서 요약/번역 기능 추가** — 실현성: 중간
   - 알PDF처럼 AI 기반 문서 요약, 외국어 문서 번역 기능 도입
   - 클라우드 API 활용 시 개발 비용 절감 가능

2. **광고 제거 일회성 구매 옵션** — 실현성: 높음
   - 월 구독 대신 $2-5 수준의 일회성 구매로 광고 제거
   - 구독 피로감 있는 사용자 타겟

3. **앱 경량화 버전 출시** — 실현성: 높음
   - 저사양 기기용 라이트 버전으로 신흥 시장 공략
   - 앱 크기 50MB 이하 목표

4. **클라우드 스토리지 연동 강화** — 실현성: 중간
   - Google Drive, OneDrive, Dropbox 직접 연동
   - 파일 동기화 및 백업 기능

5. **OCR 및 스캔 기능 강화** — 실현성: 중간
   - 종이 문서 스캔 → PDF 변환 → 텍스트 인식
   - 스캐너 앱 대체 가능
